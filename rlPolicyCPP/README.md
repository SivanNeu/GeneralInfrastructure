# RL Policy C++ Implementation

This directory contains a C++ implementation for loading and running RL policies converted from PyTorch checkpoints.

## Files

- `RLPolicyJSON.h` / `RLPolicyJSON.cpp`: JSON-based policy implementation (no external dependencies)
- `main.cpp`: Program that loads a JSON policy and computes actions
- `Makefile`: Build configuration
- `generate_test_data.py`: Python script to generate test observations and actions for validation
- `fix_checkpoint.py`: Utility script to fix PyTorch checkpoint formats (optional)

## Prerequisites

- C++17 compiler (g++ or clang++)
- No external dependencies (uses standard library only)

## Building

### Build the program:
```bash
make rlPolicyCPP
```

### Clean build artifacts:
```bash
make clean
```

## Usage

### Step 1: Convert .pth to JSON

First, convert your PyTorch checkpoint to JSON format:

```bash
cd /home/valentin/RL/src
python pth2json.py <path_to_checkpoint.pth> <output.json>
```

### Step 2: Generate Test Data (Optional, for validation)

Generate test observations and actions from the environment for validation:

```bash
python generate_test_data.py --pth=<checkpoint.pth> --env=<env_file.py> [--output=<test_data.txt>] [options]
```

Arguments:
- `--pth=<path>`: Path to PyTorch checkpoint (.pth file) - **required**
- `--env=<path>`: Path to environment Python file - **required**
- `--output=<path>`: Output file path for test data (optional, defaults to same directory as .pth file with `_test_data.txt` suffix) - **optional**
- `--samples=<N>`: Number of test observations to generate (default: 100) - **optional**
- `--method=<uniform|grid>`: Sampling method: uniform (random) or grid (default: uniform) - **optional**
- `--nonlinearity=<relu|elu|tanh>`: Activation function (must match training, default: relu) - **optional**
- `--device=<cpu|cuda>`: Device for policy inference (default: cpu) - **optional**
- `--no-actions`: Do not include actions in output file (observations only) - **optional**
- `--normalized`: Treat observations as already normalized - **optional**

Examples:
```bash
# Generate 100 test samples with actions (output file created in same directory as .pth)
python generate_test_data.py --pth=checkpoint.pth --env=point_trajectory_env.py

# Generate 200 test samples using grid sampling
python generate_test_data.py --pth=checkpoint.pth --env=point_trajectory_env.py --samples=200 --method=grid

# Generate observations only (no actions)
python generate_test_data.py --pth=checkpoint.pth --env=point_trajectory_env.py --no-actions

# Specify custom output location
python generate_test_data.py --pth=checkpoint.pth --env=point_trajectory_env.py --output=custom_path.txt
```

### Step 3: Run JSON-based policy

```bash
./bin/rlPolicyCPP --json=<jsonpath> [--test-data=<test_data.txt>] [--nonlinearity=<relu|elu|tanh>] [--normalized=<true|false>]
```

Arguments:
- `--json=<jsonpath>`: Path to JSON file (converted from .pth) - **required**
- `--test-data=<path>`: Path to test data file (generated by generate_test_data.py) - **optional**
- `--nonlinearity=<type>`: Activation type: `relu`, `elu`, or `tanh` (default: `relu`) - **optional**
- `--normalized=<bool>`: Whether input is normalized: `true` or `false` (default: `false`) - **optional**

Examples:
```bash
# Run with dummy observation
./bin/rlPolicyCPP --json=policy.json

# Run with test data for validation
./bin/rlPolicyCPP --json=policy.json --test-data=test_data.txt

# Run with specific nonlinearity
./bin/rlPolicyCPP --json=policy.json --test-data=test_data.txt --nonlinearity=elu --normalized=false
```

## Implementation Details

The implementation uses a manual neural network computation that mirrors the PyTorch model structure:

1. **Observation Normalization**: Normalizes input observations using running mean and variance
2. **Encoder**: Multi-layer perceptron (MLP) with configurable activation (ReLU, ELU, or Tanh)
3. **GRU Core**: Gated Recurrent Unit for temporal processing
4. **Distribution Head**: Linear layer that outputs mean and log-standard-deviation for action distribution

All computations are done using standard C++ without any external ML libraries, making it lightweight and easy to deploy.

## Output

When running without test data:
- Policy configuration (observation size, action size, nonlinearity, normalization)
- Forward pass results (action logits and hidden state)
- First 10 elements of action logits and hidden state (full arrays are computed)

When running with test data:
- Policy configuration
- Processing results for each test observation
- Validation summary including:
  - Number of samples processed
  - Number of samples within tolerance (if reference actions provided)
  - Maximum and average absolute differences
  - Accuracy percentage

## Notes

- All neural network operations are implemented manually in C++
